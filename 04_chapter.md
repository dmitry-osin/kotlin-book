## Глава 4. Управляющие конструкции

### 1. `if` как выражение (Expression)

Во многих языках (например, Java, C++) `if` — это инструкция (Statement). Она просто говорит программе, куда пойти, но сама по себе не может быть результатом. Поэтому там придумали тернарный оператор (`условие ? да : нет`).

В Kotlin разработчики пошли другим путем: тернарного оператора нет вообще, потому что сам **`if` умеет возвращать значение**. Это называется "выражение" (Expression).

* **Аналогия из жизни:** Покупка билета в кино. Кассир (программа) оценивает условие: «Вам есть 18 лет?». В зависимости от ответа он выдает тебе либо взрослый билет, либо детский. Сам процесс принятия решения сразу выдает готовый результат прямо тебе в руки.

```kotlin
fun main() {
    val age = 20

    // Классический подход (как в старых языках): if как инструкция
    var priceOldWay = 0
    if (age >= 18) {
        priceOldWay = 500
    } else {
        priceOldWay = 250
    }

    // Подход Kotlin: if как выражение
    // Мы сразу присваиваем результат проверки в неизменяемую переменную (val).
    // Это короче, безопаснее (нет var) и читается как обычный текст.
    val ticketPrice = if (age >= 18) 500 else 250

    // Если логика сложная, можно использовать блоки {}.
    // Последняя строчка в блоке автоматически станет результатом возврата.
    val status = if (age >= 18) {
        println("Доступ разрешен")
        "Взрослый" // Эта строка вернется в переменную status
    } else {
        println("Доступ запрещен")
        "Ребенок" // Эта строка вернется в переменную status
    }
}

```

### 2. Циклы `for` и `while`

Для повторения действий в Kotlin используются два основных цикла. Их логика интуитивно понятна.

* **Цикл `for`:** Используется, когда мы заранее знаем "маршрут" — по чему мы будем проходиться (например, от 1 до 10, или по списку покупок).
* *Аналогия:* Программа стиральной машины. Она знает, что нужно сделать ровно 5 циклов полоскания. Процесс идет по заранее известному сценарию.


* **Цикл `while`:** Используется, когда мы не знаем точного количества шагов, но знаем условие, при котором нужно остановиться.
* *Аналогия:* Мытье посуды руками. Ты не считаешь тарелки перед началом. Ты просто берешь губку и моешь тарелки *до тех пор, пока* раковина не станет пустой.



```kotlin
fun main() {
    // --- Пример цикла for ---
    // В Kotlin нет классического for(int i=0; i<10; i++)
    // Вместо этого используются диапазоны (Ranges) через оператор '..'
    
    // Пройдемся от 1 до 5 включительно
    for (i in 1..5) {
        println("Стирка: цикл $i")
    }

    // Можно шагать в обратном порядке (downTo) и через шаг (step)
    for (i in 10 downTo 0 step 2) {
        // Выведет: 10, 8, 6, 4, 2, 0
    }


    // --- Пример цикла while ---
    var dirtyPlates = 3
    
    // Цикл будет работать, пока условие внутри скобок истинно (true)
    while (dirtyPlates > 0) {
        println("Мою тарелку... Осталось: $dirtyPlates")
        dirtyPlates = dirtyPlates - 1 // Уменьшаем количество
    }
    println("Раковина чиста!")
}

```

### 3. Умный оператор `when`

Если логика ветвится не на два пути (`if-else`), а на множество, в старых языках используют конструкцию `switch-case`. В Kotlin её выкинули и создали **`when`** — один из самых мощных инструментов языка.

* **Аналогия из жизни:** Умный навигатор. Представь, что у тебя есть точка назначения. Если это соседний район — навигатор строит пеший маршрут. Если другой город — маршрут для машины по трассе. Если Турция — навигатор отключает автомобильный режим, ищет билеты на самолет и показывает путь до терминала аэропорта. Вся эта сложная логика элегантно раскидывается по пунктам назначения в одном месте, без "лапши" из десятков вложенных друг в друга `if-else`.

Оператор `when`, как и `if`, может быть выражением и возвращать значение. Ему не нужны слова `break` (в отличие от `switch`), чтобы остановить проваливание вниз — как только он находит совпадение, он выполняет код и сразу выходит.

```kotlin
fun main() {
    val destination = "Турция"

    // 1. when с передачей аргумента (проверяем конкретную переменную)
    // Результат работы when сразу сохраняется в переменную routeInfo
    val routeInfo = when (destination) {
        "Мытищи" -> "Маршрут построен. Время в пути на авто: 40 минут."
        "Турция" -> "Вылет из Шереметьево. Не забудьте загранпаспорт!"
        "Сочи", "Анапа" -> "Маршрут на юг. Возможны пробки на М4." // Можно группировать через запятую
        else -> "Направление неизвестно. Ищу варианты..." // Обязательная ветка "по умолчанию"
    }
    
    println(routeInfo)


    // 2. when БЕЗ передачи аргумента (работает как цепочка if-else)
    // Очень удобно для сложных проверок, не связанных с одной переменной.
    val speed = 110
    val isRaining = true

    when {
        speed > 130 -> println("Внимание: Превышение скорости!")
        speed > 90 && isRaining -> println("Снизьте скорость: скользкая дорога!")
        speed == 0 -> println("Остановка")
        else -> println("Полет нормальный")
    }
}

```

---

Итог:

* Конструкция `if` в Kotlin является выражением (умеет возвращать результат), что полностью устраняет потребность в тернарном операторе и позволяет писать код короче и надежнее (используя `val` вместо `var`).
* Цикл `for` используется для перебора известных коллекций или диапазонов (через оператор `..`, слова `downTo`, `step`), полностью заменяя громоздкие счетчики из C-подобных языков.
* Цикл `while` используется, когда количество итераций неизвестно, но есть четкое условие для продолжения или остановки работы.
* Оператор `when` — это эволюция классического `switch`. Он невероятно гибок: может проверять значения, работать как мощная замена множественным `if-else`, группировать условия и (как и `if`) возвращать готовый результат прямо в переменную.

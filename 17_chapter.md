## Глава 17. Infix функции

### 1. Что такое Infix-вызов?

В стандартном программировании мы привыкли отдавать команды как роботы, строго соблюдая синтаксис с точками и скобками: `объект.действие(аргумент)`.

Ключевое слово `infix` (от слова "вставлять внутрь/между") позволяет выкинуть точку и скобки, оставив только пробелы. Код превращается в классическое предложение на английском (или русском) языке: `Подлежащее Сказуемое Дополнение`.

* **Аналогия из жизни:** Дрессировка собаки.
* Классический код — это строгая команда по рации: `собака.принести(палка)`.
* Infix-вызов — это естественная человеческая речь: `собака принеси палку`. Результат один и тот же, но звучит и читается гораздо органичнее.



### 2. Встроенный пример: функция `to`

Ты наверняка уже видел эту магию в Kotlin, когда создавал словари (`Map`). Запись вида `"Ключ" to "Значение"` кажется какой-то встроенной конструкцией языка. На самом деле `to` — это не служебное слово, а обычная **инфиксная функция**!

```kotlin
fun main() {
    val city = "Москва"
    val code = 77

    // Классический вызов функции (с точкой и скобками).
    // Метод to() создает объект Pair (пару из двух значений).
    val pairOldWay = city.to(code)

    // Infix-вызов. То же самое, но без визуального "мусора".
    // Читается как текст: "Москва к 77".
    val pairInfix = city to code 

    // Именно благодаря этой функции мы так красиво создаем словари:
    val regions = mapOf("Москва" to 77, "Питер" to 78)
}

```

### 3. Как создать свою Infix-функцию?

Чтобы научить свой класс работать без точек и скобок, нужно выполнить три строгих правила:

1. Функция должна быть либо методом внутри класса, либо функцией-расширением (чтобы было "Подлежащее" слева).
2. Она должна принимать **строго один аргумент** (чтобы было "Дополнение" справа). Не два, не ноль, а ровно один.
3. Перед словом `fun` нужно добавить модификатор **`infix`**.

* **Аналогия из жизни:** Приготовление коктейля в баре. У тебя есть базовый напиток (Кофе), и ты хочешь смешать его с одной добавкой (Молоко). Инфиксная функция выступает в роли "моста" или действия между двумя ингредиентами.

```kotlin
// Класс, описывающий напиток
class Drink(val name: String) {

    // 1. Используем слово infix
    // 2. Это метод класса (Drink - слева)
    // 3. Принимает ровно один аргумент (other - справа)
    infix fun mixWith(other: Drink): Drink {
        // Создаем и возвращаем новый объединенный напиток
        return Drink("${this.name} + ${other.name}")
    }
}

fun main() {
    val coffee = Drink("Эспрессо")
    val milk = Drink("Молоко")
    val syrup = Drink("Карамель")

    // Вызываем нашу инфиксную функцию!
    // Читается как: "Кофе смешать с молоком"
    val cappuccino = coffee mixWith milk
    println(cappuccino.name) // Выведет: Эспрессо + Молоко

    // Самое крутое: инфиксные функции можно выстраивать в цепочки!
    // Это работает слева направо.
    val sweetCappuccino = coffee mixWith milk mixWith syrup
    println(sweetCappuccino.name) // Выведет: Эспрессо + Молоко + Карамель
}

```

### 4. Infix и функции-расширения

Чаще всего `infix` комбинируют с функциями-расширениями (из модуля 3.2), чтобы добавить "человекочитаемые" методы к стандартным типам (строкам, числам). Это первый шаг к написанию собственных DSL (Domain Specific Language — предметно-ориентированных языков), о которых мы поговорим чуть позже.

* **Аналогия из жизни:** Оформление билета в кино. Ты хочешь, чтобы в системе покупка ряда и места выглядела как единое предложение: `Ряд 5 место 12`.

```kotlin
// Создаем свой собственный класс для хранения координаты места
data class Seat(val row: Int, val number: Int)

// Создаем инфиксное расширение для стандартного числа (Int)
infix fun Int.seat(number: Int): Seat {
    // this - это число слева (ряд)
    // number - это число справа (место)
    return Seat(this, number)
}

fun main() {
    // Теперь мы можем создавать места невероятно красиво!
    // 5 становится "Подлежащим", seat - "Сказуемым", 12 - "Дополнением".
    val myTicket = 5 seat 12
    
    println("Ваш билет: Ряд ${myTicket.row}, Место ${myTicket.number}")
}

```

---

Итог:

* Инфиксные функции (с модификатором `infix`) позволяют вызывать методы без использования точки и круглых скобок (формат `a method b`).
* Это делает код более выразительным и читаемым, приближая его к естественному человеческому языку.
* Популярная встроенная конструкция `to` (используемая для создания словарей `mapOf(A to B)`) не является синтаксисом языка — это обычная инфиксная функция, возвращающая объект `Pair`.
* Для создания инфиксной функции необходимо соблюсти строгие правила: она должна принадлежать классу (быть методом или расширением) и принимать строго один параметр. Инфиксная функция не может принимать переменное количество аргументов (`vararg`) или использовать значения по умолчанию.

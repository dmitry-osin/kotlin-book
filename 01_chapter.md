## Глава 1. Единая система типов и Переменные (val, var)

### 1. Переменные: `val` и `var`

В Kotlin мы обязаны сразу сказать компилятору, планируем ли мы менять значение переменной в будущем. Для этого используются два ключевых слова.

* **`val` (от слова *value* — значение)** — это неизменяемая ссылка. Как только ты положил в неё данные, перезаписать их нельзя.
* *Аналогия из жизни:* `val` — это твой ИНН или дата рождения. Эти данные зафиксированы в документах один раз и навсегда. Если кто-то попытается их изменить, система выдаст ошибку.


* **`var` (от слова *variable* — переменная)** — это изменяемая ссылка. Значение внутри можно перезаписывать сколько угодно раз.
* *Аналогия из жизни:* `var` — это ценник на помидоры на рынке. Утром продавец написал там «200 рублей», к вечеру зачеркнул и написал «150 рублей». Сама картонка (переменная) осталась той же, но значение внутри изменилось.



**Золотое правило Kotlin:** Всегда по умолчанию используй `val`. Переходи на `var` только тогда, когда логика программы действительно требует изменения состояния (например, счетчик в цикле). Это защитит тебя от случайных ошибок, когда данные меняются там, где ты этого не ждешь.

```kotlin
fun main() {
    // Создаем неизменяемую переменную (val).
    // Компилятор сам догадывается, что это строка (String), благодаря автовыводу типов.
    // Почему val? Имя человека в рамках этой логики меняться не должно.
    val userName = "Алексей"
    
    // Попытка переназначить val вызовет ошибку еще до запуска программы:
    // userName = "Иван" // Ошибка компиляции: Val cannot be reassigned

    // Создаем изменяемую переменную (var).
    // Почему var? Баланс кошелька - это динамическая величина, она будет меняться.
    var walletBalance = 1000
    
    // Тратим 300 рублей. Перезаписываем значение, ошибок нет.
    walletBalance = walletBalance - 300 
}

```

### 2. Отказ от явных примитивов в синтаксисе (`Int` вместо `int`)

Если ты пришел из Java или C++, ты помнишь, что там есть два мира:

1. **Примитивы** (маленькие, быстрые, но "глупые" типы: `int`, `double`, `boolean`). У них нет методов.
2. **Объекты** (большие, тяжелые, но "умные": `Integer`, `String`). У них есть методы.

В Kotlin разработчики решили убрать это раздвоение личности на уровне написания кода. **В синтаксисе Kotlin всё является объектом.** Ты пишешь `Int` (с большой буквы), `Boolean`, `Double`.

* *Аналогия из жизни:* Представь, что ты заказываешь такси через приложение. Тебе предлагают единый интерфейс: кнопка "Заказать". Тебе как пользователю (программисту) не нужно думать, приедет машина с автоматической или ручной коробкой передач. Приложение (компилятор Kotlin) само решит, как эффективнее подать машину под капотом, но для тебя интерфейс всегда один и тот же — красивый и удобный.

Благодаря этому, прямо на числах можно вызывать полезные методы:

```kotlin
fun main() {
    val age = 30
    
    // Так как age ведет себя как объект, мы можем напрямую вызывать у него методы.
    // Например, конвертировать в строку или выполнять математические операции.
    val ageString = age.toString() 
    
    // Метод times - это то же самое, что и умножение (*).
    // Это делает язык очень гибким и позволяет в будущем переопределять операторы.
    val doubledAge = age.times(2) 
}

```

*(О том, как именно компилятор колдует с памятью, превращая этот красивый `Int` в быстрый `int` под капотом JVM, мы детально поговорим в следующем модуле 1.2).*

### 3. Иерархия типов: Any, Unit, Nothing

Раз уж всё в Kotlin — это объекты, значит, у них должна быть строгая иерархия. Как генеалогическое древо. И в этом древе есть три главных "столпа".

#### Any (Вершина мира)

`Any` — это прародитель абсолютно всех не-null типов в Kotlin. Любой класс, который ты создашь, и любой встроенный тип (например, `String` или `Int`) неявно наследуется от `Any`.

* *Аналогия:* Понятие «Материя» в физике. Камень, вода, воздух, стул — всё это материя. Если функция принимает на вход `Any`, это значит «неси сюда вообще что угодно».

#### Unit (Вежливое "Готово")

В Java/C++ функции, которые ничего не возвращают, помечаются ключевым словом `void` (пустота). В Kotlin вместо пустоты используется тип `Unit`. Это реальный объект (Singleton), который означает: «Я выполнил свою работу, но мне нечего тебе отдать в качестве результата».

* *Аналогия:* Ты просишь друга полить цветы. Он поливает их и просто кивает: «Сделано». Он не дает тебе в руки какой-то предмет, он просто сообщает о факте завершения задачи.

```kotlin
// Мы можем явно указать : Unit, но обычно это опускают (Kotlin подставит его сам)
fun printGreeting(name: String): Unit {
    println("Привет, $name!")
    // Под капотом функция возвращает объект Unit, 
    // сигнализируя, что полезного возвращаемого значения нет.
}

```

#### Nothing (Точка невозврата)

Это самый необычный тип. У класса `Nothing` вообще нет экземпляров. Если функция возвращает `Nothing`, это гарантия того, что **она никогда не завершится нормально**. Она либо выкинет ошибку (Exception), либо уйдет в бесконечный цикл.

* *Аналогия:* Дорожный знак «Тупик» или черная дыра. Если выполнение кода дошло до функции, возвращающей `Nothing`, программа дальше по этому пути не пойдет.

Зачем это нужно? Это помогает компилятору понимать логику:

```kotlin
// Эта функция всегда выбрасывает ошибку. Она никогда не вернет результат.
// Поэтому ее тип возвращаемого значения - Nothing.
fun failWithReason(reason: String): Nothing {
    throw IllegalArgumentException(reason)
}

fun processAge(age: Int) {
    // Компилятор понимает: если age < 0, вызовется failWithReason.
    // Так как failWithReason возвращает Nothing, компилятор знает, 
    // что выполнение прервется, и код ниже никогда не выполнится при age < 0.
    val validAge = if (age >= 0) age else failWithReason("Возраст не может быть отрицательным!")
    
    println("Возраст принят: $validAge")
}

```

Встроенная в Kotlin функция `TODO()` — классический пример использования `Nothing`. Она просто выбрасывает ошибку `NotImplementedError`, позволяя коду скомпилироваться, даже если функционал еще не написан.

---

Итог:

* Переменные делятся на `val` (неизменяемые ссылки, используем по умолчанию) и `var` (изменяемые ссылки, используем только при необходимости изменения состояния).
* В Kotlin нет явных примитивов на уровне написания кода: все типы (включая числа `Int`, `Double`) имеют единый объектный интерфейс и обладают встроенными методами.
* `Any` — это корневой тип (предок) абсолютно для всех классов в языке.
* `Unit` — тип, означающий, что функция успешно выполнила свою работу, но не возвращает полезных данных (современная замена устаревшему `void`).
* `Nothing` — тип, означающий "точку невозврата": функция с таким возвращаемым типом никогда не завершит свое выполнение штатно (выбросит исключение или зависнет в бесконечном цикле).

## Глава 11. Функции-расширения (Extensions)

### 1. Как это работает: Синтаксис и Receiver

Чтобы создать функцию-расширение, нужно перед именем функции написать имя класса, который мы хотим расширить. Этот класс называется **Receiver (Получатель)**.

* **Аналогия из жизни:** Представь, что ты купил базовую комплектацию автомобиля. Исходный код машины (двигатель, электроника) закрыт и запломбирован заводом на гарантии. Но тебе нужен держатель для телефона. Ты не лезешь внутрь торпеды с паяльником — ты просто покупаешь держатель на присоске и элегантно крепишь его на стекло снаружи. Машина получает новую функцию (умеет держать телефон), при этом заводская сборка осталась нетронутой.
* **Аналогия из гейминга (из нашего плана):** Это как идеальный файл конфигурации (например, `autoexec.cfg` в шутерах вроде CS2). Ты не имеешь доступа к исходникам движка игры, ты не можешь переписать логику стрельбы. Но ты можешь легко «навесить» свои собственные бинды (например, прыжок и бросок гранаты по одной кнопке) прямо поверх готовой закрытой системы.

Давай добавим стандартному классу `String` функцию, которая маскирует номер кредитной карты, оставляя только последние 4 цифры:

```kotlin
// Мы пишем String. перед именем функции. 
// String - это Receiver (получатель). Теперь эта функция "приклеена" к классу String.
fun String.maskCreditCard(): String {
    // Внутри расширения ключевое слово 'this' ссылается на саму строку, 
    // на которой была вызвана функция (как если бы мы были внутри класса String).
    
    // Если длина меньше 4, возвращаем как есть, иначе маскируем
    if (this.length <= 4) return this 
    
    val lastFourDigits = this.takeLast(4)
    val maskedPart = "*".repeat(this.length - 4)
    
    return maskedPart + lastFourDigits
}

fun main() {
    val myCard = "1234567812345678"
    
    // Магия! Мы вызываем наш метод так, будто он всегда был внутри стандартного класса String.
    // Никаких вспомогательных классов StringUtils.mask(myCard).
    val safeCardInfo = myCard.maskCreditCard() 
    
    println(safeCardInfo) // Выведет: ************5678
}

```

### 2. Расширения с дженериками (Generics)

Функции-расширения можно применять не только к конкретным классам, но и к целым семействам типов, используя обобщения (Generics).

Допустим, нам нужен безопасный метод, который достает второй элемент из любого списка. Если элементов меньше двух, он должен вернуть `null`, а не ронять программу с ошибкой.

```kotlin
// <T> означает, что функция работает с любым типом данных внутри списка.
// List<T> - это наш Receiver.
fun <T> List<T>.secondOrNull(): T? {
    // Проверяем размер списка. 'this' - это сам список.
    if (this.size >= 2) {
        return this[1] // Возвращаем элемент под индексом 1 (второй по счету)
    }
    return null
}

fun main() {
    val names = listOf("Анна", "Борис", "Виктор")
    val singleName = listOf("Анна")

    // Метод работает со списком строк
    println(names.secondOrNull()) // Выведет: Борис
    println(singleName.secondOrNull()) // Выведет: null (ошибки нет!)

    // Этот же метод автоматически работает со списком чисел
    val numbers = listOf(10, 20)
    println(numbers.secondOrNull()) // Выведет: 20
}

```

*(Кстати, вся стандартная библиотека коллекций в Kotlin, включая такие методы как `.filter()`, `.map()`, `.firstOrNull()` — это на 90% просто грамотно написанные функции-расширения поверх обычных Java-коллекций!)*

### 3. Как это выглядит «под капотом» (Важно для понимания!)

Часто новички думают, что функции-расширения реально внедряются внутрь класса и нарушают принципы ООП. Это не так.

**Расширения разрешаются статически.**
Компилятор Kotlin не меняет исходный код оригинального класса. Под капотом (в байткоде JVM) он просто создает обычную статическую функцию, куда первым аргументом передает сам объект.

Поэтому у функций-расширений есть **строгое ограничение**:

* Они **не могут** обращаться к приватным (`private`) или защищенным (`protected`) переменным и методам класса. Они "видят" класс только снаружи, точно так же, как и любой другой внешний код.

```kotlin
class BankVault(private val secretCode: Int, val address: String)

// Пытаемся написать расширение
fun BankVault.printDetails() {
    println("Адрес хранилища: ${this.address}") // Успешно, поле публичное
    
    // println("Секретный код: ${this.secretCode}") 
    // ОШИБКА КОМПИЛЯЦИИ! Расширение не может взломать инкапсуляцию и прочитать private поле.
}

```

---

Итог:

* Функции-расширения позволяют добавлять новые методы к существующим (в том числе закрытым и сторонним) классам без использования наследования и изменения исходного кода.
* Для объявления расширения перед именем функции указывается тип-получатель (Receiver), к которому она привязывается (например, `fun String.myMethod()`).
* Внутри функции-расширения ключевое слово `this` указывает на текущий экземпляр объекта-получателя.
* Расширения можно создавать для обобщенных типов (Generics), что позволяет писать универсальные утилиты для любых коллекций или структур данных.
* Под капотом функции-расширения являются статическими методами и **не имеют доступа** к приватным (`private`) и защищенным (`protected`) членам класса, тем самым полностью сохраняя инкапсуляцию ООП.

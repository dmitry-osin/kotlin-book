## Глава 2. Упаковка и распаковка примитивных типов

### 1. Как компилятор оптимизирует `Int`

По умолчанию Kotlin делает всё возможное, чтобы твой красивый объектный `Int` под капотом (на уровне виртуальной машины Java — JVM) превратился в обычный, молниеносно быстрый примитив `int`.

* **Аналогия из жизни:** Представь, что примитив (`int`) — это просто число, написанное на стикере. Он легкий, его можно быстро передать из рук в руки, он не занимает места на столе.

Компилятор Kotlin работает как строгий менеджер: если он видит, что стикера достаточно для выполнения задачи, он не будет усложнять.

```kotlin
fun main() {
    // Для нас это объект Int (мы можем вызвать age.toString()).
    // Но компилятор понимает, что число используется просто как значение.
    // В байткоде JVM это превратится в легкий примитив 'int' (стикер).
    val age: Int = 30 
    
    // Математика работает напрямую с примитивами, это очень быстро.
    val nextYearAge = age + 1 
}

```

### 2. Boxing (Упаковка): Когда примитив становится тяжелым

**Boxing (Упаковка)** — это процесс, когда легковесный примитив принудительно оборачивается в полноценный объект-обертку (в случае с JVM это класс `java.lang.Integer`).

* **Аналогия из жизни:** Упаковка — это когда ты берешь свой легкий стикер с числом и кладешь его в толстую, тяжелую картонную папку с завязками (объект `Integer`). Зачем? Потому что некоторые полки в твоем офисе (структуры данных) спроектированы так, что на них можно ставить *только* стандартные картонные папки. Голые стикеры туда класть запрещено правилами офиса.

Самый частый случай, когда компилятор вынужден делать упаковку — это **Дженерики (Обобщения)**.

Коллекции в JVM (такие как `List`, `Set`, `Map`) исторически умеют хранить только ссылки на объекты. Они не умеют работать с голыми примитивами.

```kotlin
fun main() {
    // 1. Создаем простое число (примитив 'int', без упаковки)
    val number: Int = 99 

    // 2. Создаем список чисел. 
    // List в JVM требует объектов. Компилятор невидимо для нас 
    // берет число 99 и выполняет BOXING - создает объект Integer.
    val numberList: List<Int> = listOf(99, 100, 101)

    // В памяти numberList хранит не сами числа, а ССЫЛКИ на "тяжелые папки",
    // внутри которых уже лежат числа. Это требует больше памяти (Heap) 
    // и нагружает сборщик мусора (Garbage Collector).
}

```

### 3. Unboxing (Распаковка) и способы оптимизации

**Unboxing (Распаковка)** — это обратный процесс. Когда тебе нужно достать число из коллекции (из «папки»), чтобы умножить его на 2, компилятор извлекает примитив из объекта. Эта постоянная перепаковка туда-сюда (Autoboxing) может съедать производительность в высоконагруженных системах.

**Как с этим бороться в Kotlin?**
Если тебе нужно хранить огромный массив чисел, и ты не хочешь, чтобы компилятор создал миллион "тяжелых папок" (объектов), Kotlin предлагает специальные классы-массивы для примитивов.

```kotlin
fun main() {
    // ПЛОХО ДЛЯ ПАМЯТИ: Array<Int> 
    // Это аналог Java-массива объектов (Integer[]).
    // Под капотом произойдет Boxing для каждого числа.
    val boxedArray: Array<Int> = arrayOf(1, 2, 3)

    // ОТЛИЧНО ДЛЯ ПАМЯТИ: IntArray
    // Это аналог Java-массива примитивов (int[]).
    // Компилятор хранит числа как голые примитивы. Никаких папок!
    val primitiveArray: IntArray = intArrayOf(1, 2, 3)
}

```

*Справка: Также упаковка (Boxing) гарантированно происходит, когда переменная может принимать значение отсутствия данных (null). Но механику работы с null мы детально разберем в следующей главе.*

---

Итог:

* Написание типа `Int` в Kotlin не означает, что всегда будет создаваться тяжелый объект. По умолчанию компилятор оптимизирует такие переменные в быстрые примитивы (`int` в JVM), экономя память.
* Boxing (упаковка) — это скрытое оборачивание легкого примитива в тяжелый объект (например, `Integer`).
* Упаковка чаще всего происходит при использовании дженериков (например, коллекций `List<Int>`), так как коллекции в JVM могут хранить только объекты, а не примитивы.
* Для оптимизации хранения большого количества чисел без упаковки в Kotlin следует использовать специализированные массивы примитивов, такие как `IntArray`, `DoubleArray` (вместо `Array<Int>`, `Array<Double>`).

## Глава 16. Перегрузка операторов

### 1. Переопределение `+` (plus) и `-` (minus)

Давай научим наши классы складываться.

* **Аналогия из жизни:** Раздельный счет в ресторане. Ты с другом посидел в кафе, официант принес два отдельных чека. Чтобы узнать общую сумму чаевых и расходов, вы берете два чека и физически "складываете" их вместе, получая один новый, объединенный чек.

```kotlin
// Класс, описывающий чек в кафе
class Receipt(val totalAmount: Int, val items: List<String>) {

    // Волшебное слово operator говорит компилятору: 
    // "Эта функция будет вызываться при использовании знака '+'"
    // Функция обязана называться строго 'plus'
    operator fun plus(other: Receipt): Receipt {
        // Логика сложения: суммируем деньги и объединяем списки покупок
        val newAmount = this.totalAmount + other.totalAmount
        val newItems = this.items + other.items // Списки в Kotlin тоже умеют складываться
        
        // Возвращаем НОВЫЙ объединенный чек
        return Receipt(newAmount, newItems)
    }
}

fun main() {
    val myReceipt = Receipt(500, listOf("Кофе", "Круассан"))
    val friendReceipt = Receipt(300, listOf("Чай", "Чизкейк"))

    // Магия перегрузки! Вместо myReceipt.plus(friendReceipt) мы пишем просто '+':
    val combinedReceipt = myReceipt + friendReceipt 

    println("Общая сумма: ${combinedReceipt.totalAmount}") 
    println("Все позиции: ${combinedReceipt.items}")
}

```

*(Точно так же можно переопределить вычитание — функция должна называться `minus`, умножение — `times`, и деление — `div`).*

### 2. Оператор `get` (Доступ по индексу `[]`)

В массивах или списках мы привыкли доставать элементы по индексу, используя квадратные скобки: `list[0]`. Мы можем научить наш собственный класс вести себя как коллекция.

* **Аналогия из жизни:** Поезд. Сам по себе поезд — это единый объект. Но ты можешь попросить проводника: *"Покажи мне вагон номер 3"*. Ты не просишь `train.getWagon(3)`, ты просто обращаешься к поезду по номеру: `train[3]`.

```kotlin
class Train(private val wagons: List<String>) {

    // Функция обязана называться 'get'.
    // Она может принимать не только Int (индекс), но и String, и вообще любой тип!
    operator fun get(index: Int): String {
        // Защищаемся от выхода за пределы поезда
        if (index < 0 || index >= wagons.size) {
            return "Такого вагона нет!"
        }
        return wagons[index]
    }
}

fun main() {
    val myTrain = Train(listOf("Локомотив", "Ресторан", "Купе", "Плацкарт"))

    // Вызываем оператор []. Под капотом компилятор превратит это в myTrain.get(2)
    val wagon = myTrain[2] 
    
    println("В третьем вагоне (индекс 2) находится: $wagon") // Выведет: Купе
}

```

### 3. Оператор `invoke()` (Вызов объекта как функции)

Это самый необычный и мощный оператор. Он позволяет взять объект (экземпляр класса) и поставить после него круглые скобки `()`, как будто этот объект — это функция.

* **Аналогия из жизни:** Личный помощник (Ассистент). Ты нанял опытного секретаря. Когда тебе нужно, чтобы он выполнил задачу, ты не говоришь ему: *"Ассистент, активируй свой модуль выполнения работы и передай туда задачу"*. Ты просто бросаешь ему задачу: *"Ассистент, сделай кофе!"*. Объект "Ассистент" сам знает, что его главная цель — выполнять команды.

В Kotlin это реализуется через оператор `invoke` (призыв / вызов).

```kotlin
class Assistant(val name: String) {

    // Оператор invoke. Он сработает, когда мы добавим () прямо к объекту.
    // Аргументы внутри могут быть любыми.
    operator fun invoke(task: String) {
        println("Ассистент $name приступил к задаче: '$task'")
    }
    
    // Перегружать можно сколько угодно раз с разными параметрами!
    operator fun invoke(task: String, urgency: String) {
        println("Ассистент $name выполняет '$task' со статусом: $urgency!!!")
    }
}

fun main() {
    // Создаем обычный объект
    val jarvis = Assistant("Джарвис")

    // Магия! Мы используем объект jarvis так, будто это функция.
    // Под капотом компилятор перепишет это как jarvis.invoke("Заказать пиццу")
    jarvis("Заказать пиццу") 
    
    // Вызов второй версии invoke
    jarvis("Заблокировать двери", "КРИТИЧНО") 
}

```

**Зачем это нужно?**
Оператор `invoke` массово используется при чистой архитектуре (например, в паттерне UseCase / Interactor). У тебя есть класс `GetUserInfoUseCase`, у которого ровно одна цель в жизни — достать пользователя. Вместо того чтобы писать `getUserInfoUseCase.execute()`, ты просто пишешь `getUserInfoUseCase()`. Код становится невероятно чистым и читаемым.

---

Итог:

* В Kotlin стандартные операторы (`+`, `-`, `*`, `[]`, `()`) являются "синтаксическим сахаром" — скрытыми вызовами обычных функций.
* Чтобы научить свой класс реагировать на операторы, нужно написать функцию со специальным именем (например, `plus` для `+`, `get` для `[]`) и обязательно пометить её ключевым словом **`operator`**.
* Оператор `+` (функция `plus`) позволяет интуитивно объединять объекты, делая код похожим на математику.
* Оператор `[]` (функция `get`) позволяет обращаться к внутренним данным объекта по индексу или ключу, превращая класс в подобие коллекции или словаря.
* Оператор `invoke()` позволяет обращаться к самому объекту так, будто он является функцией (`myObject(args)`), что идеально подходит для классов, выполняющих строго одно главное действие (паттерн Command или UseCase).

## Глава 13. Синонимы типов (typealias)

### 1. Что такое `typealias`?

Ключевое слово `typealias` позволяет придумать короткое, понятное и логичное "прозвище" (ярлык) для любого существующего, но слишком длинного или сложного типа данных.

* **Аналогия из жизни:** Записная книжка в телефоне. Тебе не нужно каждый раз вспоминать и набирать сложный 11-значный набор цифр `+7 (999) 123-45-67`. Ты один раз создаешь для него ярлык — «Мама» или «Начальник». Когда ты нажимаешь на ярлык, телефон всё равно звонит по оригинальному номеру. Ярлык не создает нового абонента, он просто упрощает доступ к старому.

Синонимы типов объявляются на самом верхнем уровне файла (вне классов и функций).

```kotlin
// Создаем короткий и понятный ярлык для функции, 
// которая принимает число и возвращает логическое значение (удовлетворяет ли число правилу).
typealias Rule = (Int) -> Boolean

// Теперь вместо страшной записи мы используем красивое слово Rule
fun filterNumbers(numbers: List<Int>, rule: Rule): List<Int> {
    val result = mutableListOf<Int>()
    for (num in numbers) {
        // Применяем переданное правило к каждому числу
        if (rule(num)) {
            result.add(num)
        }
    }
    return result
}

fun main() {
    val myNumbers = listOf(1, 2, 3, 4, 5, 6)

    // Передаем логику (лямбду). Компилятор знает, что Rule — это (Int) -> Boolean.
    val evenNumbers = filterNumbers(myNumbers) { it % 2 == 0 } 
    val numbersGreaterThanThree = filterNumbers(myNumbers) { it > 3 }

    println(evenNumbers) // Выведет: [2, 4, 6]
}

```

### 2. Применение для сложных структур данных

`typealias` работает не только с функциями. Его обожают использовать для длинных и вложенных коллекций.

* **Аналогия из жизни:** Название должности. В официальном договоре может быть написано: *"Старший специалист отдела обеспечения контроля качества цифровых продуктов"*. Но на офисном бейджике и в рабочих чатах используется короткий синоним: *"Тестировщик"*. Суть работы человека от этого не меняется, но общаться становится в разы проще.

```kotlin
// Представим, что мы храним настройки пользователей в виде сложного словаря:
// Ключ - ID пользователя (String), Значение - еще один словарь с настройками.
// Писать Map<String, Map<String, Boolean>> по всему коду очень неудобно.

// Создаем синоним:
typealias UserSettings = Map<String, Map<String, Boolean>>

// Теперь сигнатуры наших функций выглядят так, будто мы читаем обычный текст:
fun saveSettings(settings: UserSettings) {
    println("Настройки сохранены: $settings")
}

fun loadSettings(): UserSettings {
    // Возвращаем реальную мапу, компилятор сам подставит нужный тип
    return mapOf(
        "user_123" to mapOf("darkMode" to true, "notifications" to false)
    )
}

```

### 3. Важное архитектурное правило: `typealias` не создает новых типов!

Это ключевое отличие `typealias` от создания нового класса (`class` или `data class`).
Синоним типа существует **только на этапе написания кода**. Как только ты нажимаешь кнопку "Скомпилировать", компилятор Kotlin проходится по всему проекту, стирает все красивые ярлыки и подставляет на их место оригинальные "страшные" типы.

Это значит, что оригинальный тип и его синоним **абсолютно взаимозаменяемы**.

```kotlin
typealias Password = String
typealias Email = String

fun registerUser(email: Email, pass: Password) {
    println("Регистрация: $email")
}

fun main() {
    // Поскольку Email и Password - это просто синонимы для String, 
    // компилятор не защитит нас, если мы перепутаем их местами!
    
    val myPass = "qwerty12345"
    val myEmail = "test@mail.com"

    // Ошибки компиляции не будет, хотя логически это баг:
    registerUser(myPass, myEmail) 
}

```

*Если тебе нужна строгая защита от таких ошибок (чтобы пароль нельзя было случайно передать вместо email), `typealias` не подойдет. Для этого в Kotlin используются Value/Inline классы, которые мы детально разберем в Модуле 5.4.*

---

Итог:

* `typealias` позволяет создавать короткие, читаемые синонимы (ярлыки) для длинных и сложных типов данных.
* Идеально подходит для упрощения сложных функциональных сигнатур (например, замена `(Int, String) -> Boolean` на `Validator`) и многоуровневых коллекций.
* Объявление `typealias` происходит на уровне файла (вне классов).
* Синонимы типов не создают новые классы в памяти. На этапе компиляции они полностью заменяются оригинальными типами.
* Из-за взаимозаменяемости `typealias` не обеспечивает дополнительной строгой типизации (например, `typealias Name = String` не запретит передать в функцию любую другую обычную строку).

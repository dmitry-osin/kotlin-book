## Глава 14. Scope-функции

### 1. `apply` (Настройка объекта)

* **Внутри:** `this` (имя можно опускать)
* **Возвращает:** Сам объект
* **Аналогия из жизни:** Покупка и настройка автомобиля в салоне. Ты купил машину (создал объект), сел **внутрь** нее (`this`) и начинаешь настраивать под себя: поправил зеркало, отодвинул сиденье, включил радио. Закончив настройку, ты выезжаешь из салона на **этой же самой машине** (функция возвращает сам объект).

Идеально подходит для конфигурации объектов сразу после их создания.

```kotlin
class Car {
    var color: String = "Белый"
    var engineState: Boolean = false
    fun start() { engineState = true }
}

fun main() {
    // Без apply нам пришлось бы писать: 
    // val myCar = Car()
    // myCar.color = "Черный"
    // myCar.start()

    // С использованием apply:
    val myCar = Car().apply {
        // Мы находимся ВНУТРИ объекта (this). Слово myCar писать не нужно!
        color = "Черный" 
        start()          
    } // apply возвращает настроенную машину, и мы сохраняем ее в myCar
}

```

### 2. `also` (Дополнительное действие / Сайд-эффект)

* **Внутри:** `it` (мы смотрим на объект со стороны)
* **Возвращает:** Сам объект
* **Аналогия из жизни:** Покупка кофе. Ты взял стакан кофе (`it`), и просишь баристу: *"А сделай **заодно** (also) надпись с моим именем на стакане"*. Бариста делает пометку маркером (сайд-эффект) и возвращает тебе **тот же самый стакан**.

Идеально для логирования, проверок или записи в базу данных, когда мы не хотим прерывать цепочку вызовов.

```kotlin
fun main() {
    val prices = mutableListOf(100, 200, 300)

    // Добавляем элемент и ЗАОДНО (also) печатаем лог в консоль.
    // Внутри also список доступен как 'it'
    prices
        .apply { add(400) } // Настроили: добавили элемент (вернулся список)
        .also { println("Лог: в списке теперь элементы $it") } // Логируем (вернулся список)
        .also { require(it.size == 4) } // Проверяем (вернулся список)
}

```

### 3. `let` (Трансформация и Безопасная работа с `null`)

* **Внутри:** `it`
* **Возвращает:** Результат выполнения последней строки (лямбды)
* **Аналогия из жизни:** Почтовый ящик и посылка. Ты достаешь посылку (`it`), распаковываешь ее, собираешь из деталей стул и ставишь стул в комнату. В ящик ты положил коробку (исходный объект), а на выходе получил стул (результат).
А еще `let` — лучший друг для работы с `null`. *"Если (let) посылка вообще пришла, тогда распакуй ее"*.

```kotlin
fun main() {
    val userName: String? = "Алексей" // Может быть null

    // Комбинация безопасного вызова (?.) и let.
    // Код внутри скобок выполнится ТОЛЬКО если userName не null.
    val greetingLength = userName?.let {
        // 'it' здесь — это уже гарантированно не-null строка "Алексей"
        val greeting = "Привет, $it!"
        println(greeting)
        
        // let возвращает результат последней строчки кода.
        // Мы возвращаем длину строки (Int), и она записывается в greetingLength.
        greeting.length 
    }
}

```

### 4. `with` (Группировка вызовов)

* **Внутри:** `this`
* **Возвращает:** Результат последней строки
* **Аналогия из жизни:** Работа с документом. Ты кладешь документ на стол и говоришь: *"**Используя** (with) этот документ, прочитай заголовок, поставь печать и выдай мне справку"*.
В отличие от других scope-функций, `with` — это не функция-расширение, объект передается в нее как обычный аргумент.

```kotlin
class Server(val ip: String, val port: Int) {
    fun connect() = "Подключено к $ip:$port"
}

fun main() {
    val myServer = Server("192.168.1.1", 8080)

    // Передаем объект внутрь with
    val connectionStatus = with(myServer) {
        // Мы находимся внутри (this). Обращаемся к свойствам напрямую.
        println("Инициализация сервера $ip...")
        // Последняя строка становится результатом (String)
        connect() 
    }
    
    println(connectionStatus)
}

```

### 5. `run` (Комбинация `let` и `with`)

* **Внутри:** `this`
* **Возвращает:** Результат последней строки
* **Аналогия из жизни:** Аренда профессиональной кухни. Ты заходишь на кухню (`this`), используешь все ножи, плиты и духовки, готовишь торт, и выносишь в зал клиенту готовый торт (результат), а не саму кухню.

Работает точно так же, как `let` (возвращает результат), но внутри мы обращаемся к объекту как `this` (без имени), что делает код чище, если мы вызываем много внутренних методов.

```kotlin
fun main() {
    val text: String? = "kotlin is awesome"

    // Используем ?.run для безопасного выполнения (если text не null)
    val formattedResult = text?.run {
        // Мы внутри строки (this). Вызываем методы строки напрямую!
        val upper = uppercase() // вместо it.uppercase()
        val replaced = upper.replace("IS", "ИС")
        
        // Возвращаем результат
        "Результат работы: $replaced"
    }

    println(formattedResult)
}

```

---

Итог:

* Scope-функции создают временную область видимости для объекта, позволяя писать компактный и читаемый код без постоянного повторения имени переменной.
* **`apply`** (внутри `this`, возвращает объект) — идеален для первичной настройки и инициализации объекта после его создания.
* **`also`** (внутри `it`, возвращает объект) — используется для добавления "сайд-эффектов", таких как логирование или валидация, не разрывая цепочку вызовов.
* **`let`** (внутри `it`, возвращает новый результат) — чаще всего применяется в связке с безопасным вызовом `?.` для выполнения кода только для не-`null` объектов и трансформации данных.
* **`with`** (принимает объект как аргумент, внутри `this`, возвращает новый результат) — используется для удобной группировки множества вызовов к одному объекту, когда результат нужен в виде новых данных.
* **`run`** (внутри `this`, возвращает новый результат) — гибрид, удобный для выполнения сложных вычислений внутри контекста объекта (в том числе Nullable) с возвратом вычисленного значения.

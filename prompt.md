#### Модуль 3: Функциональное программирование (Уровень: Средний)
Инструменты для создания лаконичного, читаемого и выразительного кода.


 * **3.3. Functional Interfaces (SAM)**
   * fun interface и SAM-conversion. Замена анонимным объектам при строгих контрактах.
 * **3.4. Синонимы типов (typealias)**
   * Создание читаемых ярлыков для сложных сигнатур (например, typealias Rule = (Int) -> Boolean).
 * **3.5. Scope-функции**
   * Выполнение блоков кода в контексте объекта: let, run, with, apply, also.
 * **3.6. Коллекции и Sequence**
   * Разница между энергичными вычислениями и ленивыми стримами данных.
   * **Генераторы sequence { yield() }:** Пошаговая выдача данных с заморозкой функции. Как торговый автомат: нажал кнопку — получил одну банку (yield).
 * **3.7. Перегрузка операторов**
   * Переопределение +, -, get и оператор invoke().
#### Модуль 4: Метапрограммирование и Выразительность (Уровень: Продвинутый)
Создание собственных DSL и управление поведением компилятора.
 * **4.1. Infix функции**
   * Вызов методов без точек и скобок (a to b).
 * **4.2. Lambdas with Receivers**
   * Лямбды, выполняющиеся в контексте ресивера (this).
 * **4.3. Создание DSL (Domain Specific Language)**
   * Написание генераторов конфигураций (HTML, SQL, PDF-рендеры), читающихся как обычный текст.
 * **4.4. Gradle Kotlin DSL (.kts)**
   * Использование статической типизации Kotlin для скриптов сборки.
 * **4.5. Contracts и Context Receivers**
   * Обучение смарт-каста бизнес-логике и инъекция контекста в функции.
#### Модуль 5: Под капотом JVM: Типы, Память и Рефлексия (Уровень: Эксперт)
Оптимизация производительности и глубокое понимание байткода.
 * **5.1. Продвинутые Generics**
   * Ковариантность (out) и контравариантность (in).
 * **5.2. Стирание типов (Type Erasure) и reified**
   * Проброс знания о дженериках в рантайм.
 * **5.3. Оптимизация памяти: inline, noinline, crossinline**
   * Избежание аллокации лямбд в Heap.
   * *Аналогия:* Вызов лямбды — отправка курьера с рецептом на кухню (трата ресурсов). inline — намертво приклеенный рецепт к столу повара.
 * **5.4. Value / Inline классы**
   * Строго типизированные обертки с нулевой стоимостью памяти в рантайме.
 * **5.5. Делегирование**
   * Ленивая инициализация (by lazy) и делегирование реализации интерфейсов.
 * **5.6. Рефлексия (kotlin-reflect)**
   * Динамический анализ классов. Отличие KClass от Class.
 * **5.7. Хвостовая рекурсия (tailrec)**
   * Переписывание рекурсии в быстрый цикл для защиты от StackOverflowError.
#### Модуль 6: Асинхронность и Высокие нагрузки: Coroutines (Уровень: Эксперт)
Главный инструмент для Fintech и HighLoad. Переход к неблокирующей модели.
 * **6.1. Механика suspend функций**
   * Конечные автоматы (State Machines) и Continuation Passing Style под капотом. Сравнение с Thread.
 * **6.2. CoroutineContext и Dispatchers**
   * Управление пулами потоков (IO, Default, Main).
 * **6.3. Structured Concurrency и Кооперативная многозадачность**
   * Иерархия выполнения, Job, SupervisorJob, каскадная отмена.
   * **Кооперативная отмена (yield()):** Как не заблокировать диспетчер. *Аналогия:* Вежливый спикер на дебатах, делающий паузу (yield), чтобы проверить, не пора ли ему уйти.
 * **6.4. Примитивы синхронизации**
   * Отказ от synchronized. Использование Mutex, Семафоров и паттерна Actor.
 * **6.5. Реактивные потоки (Flow)**
   * Холодные стримы, трансформация и управление контекстом.
 * **6.6. StateFlow и SharedFlow**
   * Горячие потоки. Трансляция событий и механизмы Backpressure.
 * **6.7. Каналы (Channels)**
   * Модель CSP. Паттерны Inbox/Outbox и воркер-пулы на корутинах.
#### Модуль 7: Продвинутая работа с данными (Уровень: Хардкор)
Функциональные подходы к надежности систем.
 * **7.1. Отказ от Exceptions: Result<T>**
   * Функциональная обработка успеха и ошибки через цепочки onSuccess/onFailure.
 * **7.2. Библиотека kotlinx.serialization**
   * Сверхбыстрая сериализация на этапе компиляции (без Reflection).
 * **7.3. Строгое FP с Arrow.kt**
   * Мощные типы Either, Option, Validated и Оптика (Optics/Lenses).
#### Модуль 8: Java Interop (Уровень: Enterprise Мастер)
Бесшовная интеграция с существующим кодом.
 * **8.1. Платформенные типы (Type!)**
   * Как Kotlin воспринимает типы из Java без аннотаций Nullability и почему это опасно.
 * **8.2. Аннотации взаимодействия с байткодом**
   * @JvmStatic, @JvmOverloads, @JvmField, @JvmName, @Throws.
 * **8.3. Управление дженериками**
   * Использование @JvmWildcard и @JvmSuppressWildcards для DI-контейнеров.
 * **8.4. Взаимодействие с Java SAM**
   * Автоматическая конвертация функциональных типов.
#### Модуль 9: Архитектура и Экосистема (Уровень: Архитектор)
Интеграция языка в боевой продакшен.
 * **9.1. Кодогенерация**
   * Разница в архитектуре между быстрыми плагинами KSP и устаревшим KAPT.
 * **9.2. Kotlin + Spring Boot / Hibernate**
   * Настройка инфраструктуры: плагины kotlin-spring (all-open) и kotlin-jpa (no-arg).
 * **9.3. Ktor Framework**
   * Создание чистых микросервисов, построенных исключительно на корутинах.
 * **9.4. Kotlin Multiplatform (KMP)**
   * Шаринг бизнес-логики: написание ядра один раз для JVM, iOS, Android и WebAssembly.</T>

## Глава 10. Функции высшего порядка

### 1. Передача функции как аргумента

В обычных языках мы передаем в функции только «мертвые» данные (числа, строки, объекты). Но иногда нам нужно передать само **поведение** (кусок логики).

В Kotlin тип функции записывается так: `(ТипВхода) -> ТипВыхода`.

* **Аналогия из жизни:** Служба доставки. Ты передаешь курьеру посылку (это обычные данные — адрес и вес). Но ты также можешь передать ему **инструкцию**, что сделать у двери. Для одного заказа инструкция: *"позвонить в звонок и ждать"*, для другого — *"оставить у двери и отправить СМС"*. Курьеру (основной функции) всё равно, что именно делать у двери. Он просто доходит до нужной точки и *вызывает* ту инструкцию (функцию), которую ты ему передал.

```kotlin
// Функция высшего порядка. 
// Она принимает данные (адрес) и ДРУГУЮ функцию (actionAtDoor).
// Тип () -> Unit означает: "Функция не принимает аргументов и ничего не возвращает".
fun deliverPackage(address: String, actionAtDoor: () -> Unit) {
    println("Курьер едет по адресу: $address")
    println("Курьер прибыл на место.")
    
    // Курьер выполняет переданную ему инструкцию
    actionAtDoor() 
    
    println("Доставка завершена.\n")
}

fun main() {
    // Вариант 1: Обычная доставка. 
    // Мы передаем логику в фигурных скобках (это называется лямбда-выражение).
    deliverPackage("ул. Пушкина, д. 10", { 
        println("-> Звоню в дверь, отдаю посылку в руки.") 
    })

    // Вариант 2: Бесконтактная доставка.
    // Синтаксический сахар Kotlin: если функция (лямбда) является ПОСЛЕДНИМ аргументом, 
    // ее можно вынести за круглые скобки. Это делает код очень красивым.
    deliverPackage("пр. Ленина, д. 5") {
        println("-> Оставляю у двери, делаю фото, ухожу.")
    }
}

```

Посмотрим на пример с передачей данных внутрь самой функции-аргумента:

```kotlin
// Здесь функция-аргумент принимает на вход цену (Double) и возвращает новую цену (Double).
fun processPayment(amount: Double, calculateDiscount: (Double) -> Double) {
    // Вызываем переданную формулу расчета
    val finalPrice = calculateDiscount(amount)
    println("К оплате: $finalPrice руб.")
}

fun main() {
    // Внутри лямбды слово 'it' автоматически обозначает единственный входящий аргумент (amount)
    processPayment(1000.0) { it * 0.9 } // Скидка 10%
    processPayment(1000.0) { it - 500 } // Скидка ровно 500 рублей
}

```

### 2. Возврат функции как результата

Функция может не только принимать логику, но и создавать её, отдавая в качестве результата (возвращать функцию).

* **Аналогия из жизни:** Автомат по продаже инструментов. Ты бросаешь монетку и нажимаешь кнопку "Молоток" или "Отвертка". Автомат не забивает гвоздь за тебя (он не выдает готовый результат). Автомат выдает тебе **инструмент** (функцию), который ты положишь в ящик и будешь использовать позже там, где тебе нужно.

```kotlin
// Функция высшего порядка, которая ВОЗВРАЩАЕТ другую функцию.
// Ее тип возвращаемого значения: (Double) -> Double
fun getTaxCalculator(countryCode: String): (Double) -> Double {
    // В зависимости от страны, мы конструируем и возвращаем разные "инструменты" расчета
    return when (countryCode) {
        "RU" -> { income -> income * 0.13 } // Возвращаем лямбду для налога 13%
        "US" -> { income -> income * 0.30 } // Возвращаем лямбду для налога 30%
        "UAE" -> { income -> 0.0 }          // В Дубае налогов нет, возвращаем всегда 0
        
        // Если страна неизвестна, возвращаем функцию, которая отдает доход без изменений
        else -> { income -> income } 
    }
}

fun main() {
    val mySalary = 100_000.0

    // 1. Получаем "инструмент" для России. 
    // Обрати внимание: мы еще ничего не посчитали! 
    // Переменная russianTaxCalc теперь сама является функцией.
    val russianTaxCalc = getTaxCalculator("RU")
    
    // 2. Получаем "инструмент" для США
    val usTaxCalc = getTaxCalculator("US")

    // 3. Теперь мы можем использовать эти инструменты сколько угодно раз
    println("Налог в РФ: ${russianTaxCalc(mySalary)}")
    println("Налог в США: ${usTaxCalc(mySalary)}")
}

```

---

Итог:

* В Kotlin функции являются объектами первого класса: их можно сохранять в переменные, передавать в другие функции и возвращать как результат работы.
* **Функция высшего порядка** — это любая функция, которая принимает другую функцию как аргумент или возвращает функцию.
* Сигнатура типа функции записывается в виде `(ТипАргумента1, ТипАргумента2) -> ТипРезультата`. Если аргументов нет, пишутся пустые скобки `()`. Если функция ничего не возвращает, указывается `Unit`.
* Если функция (лямбда-выражение) передается в качестве последнего аргумента, в Kotlin принято выносить её за пределы круглых скобок вызова для улучшения читаемости кода.
* Если функция принимает только один аргумент, внутри лямбда-выражения к нему можно неявно обращаться по ключевому слову `it`.
